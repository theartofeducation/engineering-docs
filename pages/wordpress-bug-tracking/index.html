    WordPress Bug Tracking

    1. Any user can report bugs using the Report a Bug form
    2. The reported issue will follow the process described in the Bug Triage Process document
      • Other teams can communicate regarding bugs directly through the #productmanagement channel on Slack
    3. Product and QA assess the priority of the bug based on how business is affected
    4. Prioritized bugs are transferred into Clubhouse. These are added into the WordPress project and prioritized and labeled appropriately
    5. Based on priority, they are added to the following sprint, unless the bug is determined to be a Blocker. In this instance, they are pulled in right away for work to begin
    6. Once Happy Medium has created a fix, they will push it to the staging server and their Quality Assurance Engineer follows the QA Process
    7. Once the work passes QA, the fix is submitted to the Stakeholder for approval
    8. Once the fix has been approved by Software Engineering and the cardâ€™s Stakeholders, it will push the fix to the production server
